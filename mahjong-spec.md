## 1. 專案概述

### 1.1 目標
開發一款純前端的台灣16張麻將網頁單機遊戲，玩家與三位 AI 對打，體驗真實的台灣麻將玩法。

### 1.2 核心需求
| 需求 | 說明 |
|------|------|
| 完整規則 | 台灣16張麻將標準規則 |
| AI 對手 | 三種難度：簡單、普通、困難 |
| 拖牌操作 | 直覺的拖放式出牌 |
| 胡牌判定 | 即時、準確的胡牌檢測 |

### 1.3 目標平台
- 瀏覽器（Chrome, Firefox, Safari, Edge）
- 響應式設計，支援桌面與平板

## 2. 遊戲規則

### 2.1 牌組組成

#### 牌張數量
| 牌種 | 數量 |
|------|------|
| 萬子（一萬～九萬） | 每種4張，共36張 |
| 索子（一索～九索） | 每種4張，共36張 |
| 筒子（一筒～九筒） | 每種4張，共36張 |
| 風牌（東南西北） | 每種4張，共16張 |
| 箭牌（中發白） | 每種4張，共12張 |
| **總計** | **136張** |

#### 牌面顯示
- 使用 Unicode 麻將字元或自製 SVG/CSS 牌面
- 牌面需清晰辨識，支援中文顯示

### 2.2 遊戲流程

```
洗牌 → 砌牌 → 擲骰 → 開始 → 循環（摸牌→出牌）→ 結束
```

#### 詳細流程
1. **洗牌**：隨機打亂136張牌
2. **砌牌**：每位玩家面前碼放17墩（17×2=34張），共四排
3. **莊家**：第一局隨機決定，之後輪莊或連莊
4. **發牌**：莊家拿17張，其餘玩家各拿16張
5. **出牌**：莊家先打出一張，遊戲開始

### 2.3 操作規則

#### 基本動作
| 動作 | 說明 |
|------|------|
| 摸牌 | 從牌牆摸一張牌 |
| 打牌 | 選擇一張手牌打出 |
| 吃 | 用上家打出的牌組成順子 |
| 碰 | 用任何玩家打出的牌組成刻子 |
| 槓 | 用任何玩家打出的牌或手牌組成槓子 |
| 胡 | 達成胡牌條件，結束牌局 |

#### 優先級規則
```
胡 > 槓 > 碰 > 吃
```
當多位玩家同時喊牌時：
- 胡牌優先級最高
- 槓次之
- 碰再次之
- 吃最低（只能吃上家的牌）

### 2.4 胡牌條件

#### 基本型態
台灣16張麻將胡牌 = **5組面子 + 1對將** 或 **特殊牌型**

```
標準胡牌：(AAA/ABC) × 5 + AA = 17張（含胡的那張）
```

#### 面子組合
| 類型 | 說明 | 範例 |
|------|------|------|
| 順子 | 同花色連續三張 | 一二三萬、四五六索 |
| 刻子 | 相同三張 | 三個東風、三個五筒 |
| 槓子 | 相同四張 | 四個七萬（明/暗槓） |

#### 特殊牌型
| 牌型 | 台數 | 說明 |
|------|------|------|
| 天胡 | 16台 | 莊家起手胡牌 |
| 地胡 | 16台 | 非莊家第一輪胡牌 |
| 人胡 | 8台 | 第一輪內胡牌 |
| 大四喜 | 16台 | 東南西北各一刻 |
| 大三元 | 8台 | 中發白各一刻 |
| 字一色 | 8台 | 全為字牌 |
| 清一色 | 8台 | 全為同一花色 |
| 混一色 | 4台 | 同一花色＋字牌 |
| 碰碰胡 | 4台 | 全為刻子（無順子） |
| 全求人 | 2台 | 只剩一張單吊，全部靠吃碰 |
| 湘槓尾 | 1台 | 槓上開花 |
| 自摸 | 1台 | 自己摸牌胡牌 |
| 門清 | 1台 | 沒有吃碰槓 |
| 平胡 | 1台 | 全順子無刻子 |
| 海底撈月 | 1台 | 最後一張摸牌胡牌 |
| 河底撈魚 | 1台 | 最後一張打出被胡 |
| 槓上開花 | 1台 | 槓牌後摸牌胡牌 |
| 搶槓胡 | 1台 | 搶別人的明槓胡牌 |

> **註**: 台數計算以實際規則為準，以上為參考值，正式規格將在研究完成後修正。

### 2.5 流局條件
1. 牌牆摸完無人胡牌
2. 四風連打（開局連續打同一風牌四次）
3. 四家連續打出同一張牌
4. 四槓流局（槓超過四個）

## 6. AI 系統設計

### 6.1 雙模式架構

本遊戲支援兩種 AI 驅動模式，可透過設定自由切換：

```
AI 引擎
├── 模式A：演算法 AI（純本地，離線可玩，完全免費）
└── 模式B：LLM AI（API 驅動，更像人類決策）
```

### 6.2 模式A：演算法 AI

#### 難度層級

##### 簡單（Easy）
- **策略**: 隨機 + 基本效率
- **出牌**: 隨機打出「無用牌」
- **吃碰槓**: 看到能用就用
- **胡牌**: 只判斷是否能胡
- **思考時間**: 0.5-1 秒

##### 普通（Normal）
- **策略**: 手牌效率優化
- **出牌**: 計算向聽數，打出最差牌
- **吃碰槓**: 評估效率後決定
- **防守**: 基本的危險牌判斷
- **思考時間**: 1-2 秒

##### 困難（Hard）
- **策略**: 攻防平衡 + 讀牌
- **出牌**: 綜合效率 + 防守
- **吃碰槓**: 戰略性決定
- **防守**: 完整的危險牌分析
- **讀牌**: 根據捨牌堆推測對手手牌
- **思考時間**: 2-3 秒

### 6.3 模式B：LLM AI

使用 LLM API 驅動 AI 決策，決策更像人類，並可支援對話式互動。

#### 支援的 Provider

| Provider | 模型 | 說明 |
|----------|------|------|
| **MiniMax** | M2.7 / M2.5 | Jason 主要使用的 API |
| **OpenRouter** | 多種模型（GPT-4, Claude, etc.） | 統一路由 |
| **Gemini** | Gemini Pro / Flash | Google 生態 |

#### LLM AI 設定結構

```typescript
interface LLMConfig {
  provider: 'minimax' | 'openrouter' | 'gemini';
  model: string;           //  provider 指定的模型名稱
  apiKey: string;          //  對應的 API Key
  baseUrl?: string;        //  客製端點（可選）
  temperature?: number;    //  創意程度（預設 0.7）
  thinkingTime?: number;   //  模擬思考延遲（秒）
}

interface AIConfig {
  mode: 'algorithm' | 'llm';
  algorithmDifficulty?: 'easy' | 'normal' | 'hard';  // 演算法模式時的難度
  llm?: LLMConfig;                               // LLM 模式時的設定
}
```

#### LLM 決策流程

```
1. 收集上下文
   ├── 自己的手牌（已排序）
   ├── 場上的捨牌（各家的）
   ├── 吃碰槓狀態
   ├── 牌牆剩餘數量
   └── 目前的台數

2. 組建 Prompt（送給 LLM）
   └── 請見下方 Prompt 範本

3. 呼叫 LLM API

4. 解析回應 → 動作（打牌/吃/碰/槓/胡）

5. 如有需要，顯示 LLM 的思考理由（可選）
```

#### LLM Prompt 範本

```
你是專業的台灣16張麻將玩家。

【當前局面】
- 你的位置：東/南/西/北
-你的手牌：[一萬 二萬 三萬 五索 六索 七筒]
- 場上捨牌：...[省略]...
- 牌牆剩餘：XX 張
- 目前的台數：X 台

【請選擇動作】
請根據麻將策略，選擇：
1. 打哪張牌（從手牌中選一張打出）
2. 是否要吃/碰/槓（如果場上有適合的牌）
3. 是否要胡牌（如果你已經聽牌）

請用以下格式回覆：
Action: [打/吃/碰/槓/過]
Tile: [牌名]
Reason: [你的思考理由]
```


### 6.5 演算法核心邏輯（模式A）

#### 向聽數計算
```
向聽數 = 17 - (已組合的牌數 + 最優配對數)
```

#### 牌效率評估
```
每張牌的「有效牌數」 = 還在牌牆中的能讓手牌前進的牌數
打出「有效牌數最低」的牌
```

#### 防守評估
```
危險度 = 對手可能需要此牌的機率 × 對手可能已聽牌的機率
當危險度 > 閾值時，優先打安全牌
```

## 9. 風險與挑戰

| 風險 | 影響 | 緩解措施 |
|------|------|----------|
| 胡牌判定複雜度高 | 進度延誤 | 先實作基本型態，特殊牌型後加 |
| AI 策略效果差（演算法） | 遊戲體驗差 | 參考開源麻將 AI 實作 |
| LLM API 不穩定或限流 | 遊戲中斷 | 演算法 AI 作為後備，離線可用 |
| LLM 決策延遲高 | 等待時間長 | 模擬思考時間 + 非同步載入 |
| API 費用（LLM 模式） | 持續成本 | 提示用戶注意用量，預設演算法模式 |
| 拖牌在行動裝置上不順 | 無法使用 | 備用：點擊出牌 |
| 牌面渲染效能 | 卡頓 | 考慮 Canvas 渲染 |
